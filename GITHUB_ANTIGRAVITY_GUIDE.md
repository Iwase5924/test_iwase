# GitHub & Antigravity 連携ガイド

このドキュメントでは、AIアシスタント「Antigravity」とGitHubを連携させて、効率的に開発を進める方法を解説します。

## 1. 初期設定

Antigravityがあなたの代わりにGit操作を行うためには、以下の設定が必要です。

- **ユーザー名とメールアドレス**: `git config` で設定します。
- **リモートリポジトリのURL**: `git remote add origin <URL>` で連携先を指定します。

## 2. Antigravityができること

Antigravityは、チャットを通じて以下の操作を実行できます。

- **コードの生成と修正**: 要件を伝えるだけで、プログラムの作成や既存ファイルの修正、バグ修正を行います。
- **Git操作の代行**:
  - `git add`: 変更したファイルをステージングエリアに追加します。
  - `git commit`: 変更内容に適切なメッセージを添えてコミットします。
  - `git push`: ローカルの変更をGitHubのリポジトリに反映させます。
  - `.gitignore` の作成: プロジェクトの種類に合わせて、不要なファイルを自動で除外設定します。
- **ドキュメント作成**: `README.md` や技術仕様書、操作マニュアルなどを日本語で作成します。

## 3. 認証について

Antigravityが `git push` や `git pull` を実行する際、GitHubの認証（ユーザー名とパスワード、またはパーソナルアクセストークン）が必要になる場合があります。

- **対話型認証**: ターミナルで認証を求められた場合、ユーザーが直接入力して対応する必要があります。
- **SSH / トークンの利用**: よりスムーズな連携のために、SSHキーの設定やトークンを使用した認証が推奨されます。

## 4. ベストプラクティス

1. **具体的な依頼**: 「〇〇の機能を実装してコミットまでして」と具体的に指示を与えてください。
2. **変更の確認**: Antigravityが提案した修正内容やコミットメッセージを確認してから、プッシュを依頼することをお勧めします。
3. **コミット粒度の維持**: AIに大きな修正を一気に行わせるのではなく、機能単位やタスク単位で細かく指示を出すと、Gitの履歴が管理しやすくなります。
